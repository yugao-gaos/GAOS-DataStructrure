Installation

Option 1: Unity Package Manager (Recommended)

The easiest way to install GAOS DataStructure is through Unity's Package Manager with Git URL:

https://github.com/yourusername/com.gaos.datastructure.git
  1. Open Unity and your project
  2. Go to Window > Package Manager
  3. Click the + button in the top-left corner
  4. Select Add package from git URL...
  5. Enter the URL above
  6. Click Add

Dependencies

GAOS DataStructure has a dependency on GAOS ServiceLocator. Make sure it's installed in your project before using DataStructure.

Using the Data Structure Editor

Creating a Data Structure

Start by creating a new Data Structure asset in your project:

  1. Right-click in your Project window
  2. Select Create > GAOS > Data Structure
  3. Name your new Data Structure (e.g., "PlayerData")

This creates a ScriptableObject asset that will serve as your data template.

Designing Your Data

Use the Data Structure Editor to design your data hierarchy:

  1. Select your Data Structure asset in the Project window
  2. In the Inspector, click Open Data Structure Editor
  3. Add fields using the + button
  4. For each field, specify:
    • Name (e.g., "playerName", "level", "health")
    • Type (string, int, float, etc.)
    • Default value (optional)
  5. Add nested containers for organizing related data
  6. Save your changes

Creating Instances

Create a Runtime Instance

To use your Data Structure at runtime, create an instance from your template:

// Load your Data Structure asset
var playerDataStructure = Resources.Load<DataStructure>("PlayerData");

// Create a runtime instance
var playerInstance = playerDataStructure.CreateInstance("PlayerOne");

This creates a temporary instance that exists only in memory during runtime.

Create a Persistent Asset

You can also create DataInstance assets directly from the Data Structure inspector:

  1. Select your Data Structure asset in the Project window
  2. In the Inspector, click Create Instance
  3. Name your instance and save it
  4. The new DataInstance will be created as a ScriptableObject asset

These DataInstance assets are persistent and can be referenced and used throughout your project.

In code, you can create persistent assets using (Editor only):

#if UNITY_EDITOR
// Create a persistent asset (Editor only)
var persistentInstance = playerDataStructure.CreatePersistentInstance(
    "Assets/Data/Instances", 
    "PlayerCharacter"
);
#endif

Instance vs. Structure

Key points to understand about the relationship between DataInstance and DataStructure:

  • A DataInstance is linked to its parent DataStructure
  • It only stores the values that differ from the template (overrides)
  • Changes to the parent structure will automatically propagate to instances
  • Instances can be reset to match their parent template

This template-instance pattern is memory efficient and perfect for creating variations of data.

Accessing Data

Reading Values

Access data from your instance using the Get methods:

// Get data from a runtime instance
string playerName = playerInstance.Container.Get<string>("playerName");
int level = playerInstance.Container.Get<int>("level");
float health = playerInstance.Container.Get<float>("health");

// From a DataInstance component
string playerName = instanceComponent.Container.Get<string>("playerName");

Using Path Access

Access nested data using dot notation:

// Access nested values using path notation
int strength = playerInstance.Container.PathGet<int>("stats.strength");
int agility = instanceComponent.Container.PathGet<int>("stats.agility");

Modifying Values

Update data in your instance (without affecting the template):

// Set values in your instance
playerInstance.Container.Set("playerName", "Hero");
playerInstance.Container.Set("level", 5);

// Set nested values 
playerInstance.Container.PathSet("stats.strength", 15);
